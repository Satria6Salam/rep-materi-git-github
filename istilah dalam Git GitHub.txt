>	Istilah dalam GIT
1) Branch: membuat percabangan dalam proses pembuatan code
2) Merge: penggabungan dari beberapa branch
3) Push: mengirim commit
4) pull: mengambil commit
5) repo: folder project 
6) commit: rekaman / snapshot dari repo
7) hash: penanda unik pada sebuah commit
8) checkout: berpindah ke sebuah commit 
9) remote: sumber yang memiliki repo
10) clone: mengambil repo dari remote
11) pull request: meminta pemilik repo untuk mengambil perubahan yang telah dilakukan
12) merge conflicts: ketika baris yang sama diubah oleh dua branch yang berbeda
13) fork/forking: membuat copy/duplikat dari repo orang lain beserta repository, jembatan antara repo original dan duplikatnya.
